Ветки 

Просмотреть ветки проекта — git branch

Создать ветку — git branch <название_ветки>

		Название ветки в Git может состоять из букв, цифр, а также 
		включать любой из четырёх символов: ., -, _, /. 
		Эти символы не несут особого смысла.  
		Ветки не образуют иерархии, как директории, разделённые символом /.

git branch -a — показать все известные ветки, как локальные (в локальном репозитории), 
		так и удалённые (в origin, или на GitHub).


Переключиться на другую ветку — git checkout <название_ветки>


Создать ветку и сразу переключиться на неё — git checkout -b <название_ветки>


Сравнить ветки — git diff <название_ветки1> <название_ветки2>

git diff HEAD~2 HEAD — покажи разницу между тем коммитом, который был два коммита назад, и текущим.



Выполнить слияние — git merge <название_ветки>

Удалить ветку после объединения — git branch -D <название_ветки>

		У команды git branch -D есть более безопасный вариант с флагом -d. 
		Он удалит ветку только если она была полностью объединена с другой — 
		то есть если две ветки стали (или изначально были) частью одной истории. 
		Например, если вы нечаянно создали ветку с неправильным названием, её 
		можно удалить через git branch -d %имя_ветки%. 


git push -u origin my-branch  — отправь новую ветку my-branch в удалённый репозиторий и свяжи локальную ветку 
				с удалённой, чтобы при дополнительных коммитах можно было писать 
				просто git push без -u;

git push my-branch — отправь дополнительные изменения в ветку my-branch, которая уже существует 
			в удалённом репозитории;

git pull  — подтяни изменения текущей ветки из удалённого репозитория.




Две ветки находятся в состоянии fast-forward, если одну из них можно «перемотать» вперёд и она будет содержать 
те же коммиты, что и другая.
Fast-forward слияние веток можно отключить флагом --no-ff. Например: git merge --no-ff <сливаемая ветка>. 
Также его можно отключить «навсегда» (до тех пор, пока вы не вернёте настройку «как было») с помощью 
настройки merge.ff: git config [--global] merge.ff false.

# находимся в ветке main
# --no-edit избавляет от необходимости вводить сообщение для merge-коммита
# --no-ff отключает fast-forward слияние веток
$ git merge --no-edit --no-ff сливаемая ветка
